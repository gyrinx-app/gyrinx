# Icon Audit

## 1. Complete Icon Inventory

414 total icon instances across all templates. 9 use `aria-hidden` or `aria-label`; the remaining 405 have no explicit accessible attributes.

Two syntactic formats are in use:

- **Hyphenated (dominant):** `class="bi-pencil"` -- 392 instances
- **Space-separated:** `class="bi bi-pencil"` -- 22 instances

The space-separated format appears in newer templates (`list_fighter_assign_delete_confirm.html`, `list_fighter_equipment_sell.html`, `campaign_actions.html`, `campaign_action_outcome.html`, error pages). Both are valid Bootstrap Icons syntax, but mixing them is inconsistent.

## 2. Grouping by Semantic Concept

### Add/Create (47 instances, 3 icons)

| Icon | Count | Context |
|------|-------|---------|
| `bi-plus-circle` | 19 | Add items, log actions, show stash |
| `bi-plus-lg` | 14 | "New Campaign", "New List", add injury |
| `bi-plus` | 14 | Inline add buttons (gear edit) |

**Problem:** Three different icons for the same concept. `bi-plus-circle` used for in-context additions, `bi-plus-lg` for top-level creation, `bi-plus` for inline form adds. No clear rule governs the choice.

### Edit (28 instances, 2 icons)

| Icon | Count | Context |
|------|-------|---------|
| `bi-pencil` | 27 | Edit buttons throughout |
| `bi-pencil-square` | 1 | "Edit Stats" link (fighter-edit page) |

**Problem:** Minor inconsistency. `bi-pencil-square` is used once; all other edit actions use `bi-pencil`.

### Delete/Remove (30 instances, 4 icons)

| Icon | Count | Context |
|------|-------|---------|
| `bi-trash` | 26 | Delete actions throughout |
| `bi-x-lg` | 2 | Close/cancel |
| `bi-x-circle` | 2 | Cancel/remove |
| `bi-dash-circle` | 1 | Remove item |

**Problem:** `bi-trash` is well-established for delete. However, `bi-x-lg`, `bi-x-circle`, and `bi-dash-circle` overlap with "close" and "remove" semantics.

### Warning/Error (32 instances, 3 icons)

| Icon | Count | Context |

## 4. Accessibility Audit

### Icons with Accessible Labels (9 of 414 = 2.2%)

| Template | Icon | Attribute |
|----------|------|-----------|
| `list_fighter_advancement_confirm.html` | `bi-check-circle` | `aria-hidden="true"` |
| `list_campaign_actions.html` | `bi-plus-circle` | `aria-hidden="true"` |
| `list_campaign_resources_assets.html` | `bi-arrow-left-right` | `aria-hidden="true"` |
| `list_campaign_resources_assets.html` | `bi-pencil` | `aria-hidden="true"` |
| `list_attributes.html` | `bi-pencil` | `aria-hidden="true"` |
| `list_fighter_advancement_select.html` | `bi-check-circle` | `aria-hidden="true"` |
| `dice.html` | `bi-dice-{{ i }}` | `aria-label="Dice showing {{ i }}"` |
| `list_fighter_advancement_dice_choice.html` | `bi-arrow-right` | `aria-hidden="true"` |
| `list_fighter_advancement_type.html` | `bi-arrow-right` | `aria-hidden="true"` |

**Problems:**

1. **97.8% of icons lack any accessibility attribute.** When an icon accompanies visible text (e.g., `<i class="bi-pencil"></i> Edit`), `aria-hidden="true"` should be added to the icon to prevent screen readers from reading the icon's Unicode character.
2. **Icon-only actions** (e.g., `<i class="bi-plus"></i>` without adjacent text in `list_fighter_gear_edit.html`) need `aria-label` on the parent button/link, or `role="img" aria-label="..."` on the icon itself.
3. The newer advancement templates from a single development effort are the only ones that added `aria-hidden="true"` -- indicating this wasn't a systemic practice.

| Concept | Icons Used | Recommended Canonical |
|---------|-----------|----------------------|
| Add/create | `bi-plus-circle`, `bi-plus-lg`, `bi-plus` | `bi-plus-lg` (primary), `bi-plus-circle` (contextual add with circle affordance) |
| Edit | `bi-pencil`, `bi-pencil-square` | `bi-pencil` |
| Delete | `bi-trash`, `bi-x-lg`, `bi-x-circle`, `bi-dash-circle` | `bi-trash` (delete), `bi-x-lg` (close/dismiss) |
| Warning | `bi-exclamation-triangle`, `-fill`, `bi-exclamation-circle` | `bi-exclamation-triangle` (normal), `-fill` (destructive only) |
| Info | `bi-info-circle`, `-fill` | `bi-info-circle` |
| Confirm/success | `bi-check-circle`, `bi-check-lg`, `bi-check2` | `bi-check-circle` (confirm), `bi-check-lg` (save) |
| Back | `bi-chevron-left`, `bi-arrow-left`, `bi-arrow-return-left` | `bi-chevron-left` |
| Home | `bi-house-door`, `bi-house` | `bi-house-door` |
| More options | `bi-three-dots-vertical`, `bi-three-dots` | `bi-three-dots-vertical` |
| Icon class syntax | `bi-*` and `bi bi-*` | `bi-*` (hyphenated, no prefix class) |

## 6. Recommendations

### R1: Standardise Icon Syntax

Use the hyphenated format `class="bi-icon-name"` everywhere. Migrate the 22 instances of `class="bi bi-icon-name"` to the hyphenated form. Files to update:

- `list_fighter_remove_injury.html`
- `list_fighter_state_edit.html`
- `print_config/form.html`
- `list_fighter_injuries_edit.html`
- `campaign_action_item.html`
- `list_fighter_assign_delete_confirm.html`
- `pack_item_add.html`
- `list_fighter_equipment_sell.html`
- `campaign_actions.html`
- `campaign_action_outcome.html`
- `account/login.html`
- `404.html`
- `errors/error.html`

### R2: Establish a Canonical Icon Map

Define a single source of truth (e.g., in the design system documentation or as a template include reference):

| Concept | Icon | Notes |
|---------|------|-------|
| Add (primary) | `bi-plus-lg` | Creating new entities |
| Add (contextual) | `bi-plus-circle` | Adding within an existing context |
| Edit | `bi-pencil` | All edit actions |
| Delete | `bi-trash` | Destructive deletion |
| Close/dismiss | `bi-x-lg` | Closing panels, clearing |
| Remove (inline) | `bi-dash` | Removing list items |
| Back | `bi-chevron-left` | All back navigation |
| Forward (step) | `bi-arrow-right` | Wizard "Next" steps |
| Forward (row) | `bi-chevron-right` | Mobile row navigation |
| Home | `bi-house-door` | Home navigation |
| Search | `bi-search` | Search inputs |
| Refresh | `bi-arrow-clockwise` | Update/refresh actions |
| Warning | `bi-exclamation-triangle` | General warnings |
| Warning (destructive) | `bi-exclamation-triangle-fill` | Irreversible actions only |
| Info | `bi-info-circle` | Informational callouts |
| Confirm | `bi-check-circle` | Confirmation actions |
| Save | `bi-check-lg` | Save/submit buttons |
| Copied | `bi-check2` | Clipboard copy feedback |
| More options | `bi-three-dots-vertical` | Overflow menus |
| Print | `bi-printer` | Print actions |
| Archive | `bi-archive` | Archive actions |

| User/owner | `bi-person` | User identity |
| Campaign | `bi-award` | Campaign context |
| Content pack | `bi-box-seam` | Content packs |

### R3: Add Accessibility Attributes to All Icons

1. **Decorative icons** (accompanied by visible text): Add `aria-hidden="true"` to the `<i>` element.
2. **Functional icons** (icon-only buttons/links): Add `aria-label` to the parent `<a>` or `<button>`, and `aria-hidden="true"` to the `<i>`.
3. **Informational icons** (standalone meaning, like dice): Add `role="img"` and `aria-label` to the `<i>` element.

Priority files (icon-only actions with no accessible label):

- `list_fighter_gear_edit.html` -- `bi-plus` with no text
- `fighter_card_weapon_menu.html` -- `bi-crosshair`, `bi-arrow-up-circle` icon-only links
- Multiple templates with `bi-three-dots-vertical` overflow trigger

### R4: Define Icon Sizing Tokens

Create standard sizing classes for icons:

| Token | Size | Use Case |
|-------|------|----------|
| (default) | Inherited from text | Inline with body text |
| `.icon-sm` | `fs-7` | Small/compact contexts |
| `.icon-lg` | `fs-4` | Featured/hero contexts |

Apply consistently instead of ad-hoc `fs-3`, `fs-4`, `fs-5`, `fs-6` on individual icons.

### R5: Reduce Icon Count

The codebase uses 80+ distinct icon classes. Consider whether some rarely-used icons (used once or twice) can be consolidated:

- `bi-pencil-square` (1x) -> `bi-pencil`
- `bi-house` (1x) -> `bi-house-door`
- `bi-exclamation-circle` (1x) -> `bi-exclamation-triangle`
- `bi-clock` (1x) vs `bi-clock-history` (2x) -> pick one
- `bi-list` (1x) vs `bi-list-ul` (4x) -> pick one
- `bi-arrow-return-left` (1x) -> `bi-chevron-left`
- `bi-arrow-counterclockwise` (1x) -- evaluate if `bi-arrow-clockwise` with text suffices
