Design system

Living reference for the Gyrinx design system. Font: mynor-variable (Adobe Typekit). Framework: Bootstrap 5.3 with custom SCSS overrides.

Principles

  1. Dense over spacious — show more data per screen.
  2. Colour for state only — the UI is neutral grey; colour signals state or action type.
  3. Mobile-first — design at 375px, enhance at md/lg/xl.
  4. Server-rendered — full-page loads, Django templates, standard form POST.
  5. Bootstrap vocabulary — custom classes only for gaps Bootstrap can't fill.
  6. Semantic HTML — one <h1> per page, heading levels never skip.
  7. Consistent state colours — active=success, injured/captured=warning, dead=danger.

Feedback (alerts)

Use alert alert-icon everywhere. Add/remove alert-dismissible based on context. Always include an icon as the first child.

Dismissible (flash messages after redirects)

Non-dismissible (inline contextual feedback)

This gang uses Content Packs not allowed by this Campaign: Scavvy Pack.
Starting the campaign will clone all gangs. This cannot be undone.
Remove gang from campaign?

The gang will be archived and all held assets will be unassigned.

This action will be logged to the campaign action log.
Available XP: 6 · Cost: 3 XP

This is a multi-line info note to demonstrate text alignment.

The second line stays aligned with the first, not with the icon.

Rules

Colour Icon When to use Dismissible?
alert-success bi-check-lg Action completed successfully Flash only
alert-danger bi-exclamation-triangle Errors, validation failures, blocked actions Both
alert-warning bi-exclamation-triangle Destructive confirmations, irreversible changes, cautions Inline only
alert-info bi-info-circle Contextual hints, "this will be logged", next-step guidance Both
alert-secondary bi-info-circle Neutral meta info (XP display, error IDs) Inline only
